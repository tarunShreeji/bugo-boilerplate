Introducing Bugo

Bugo came out of a need to develop websites quickly. Options to build a website with blogging capabilities were limited. Systems like WordPress, Joomla and others require intimate knowledge of them in order to develop anything custom out of the box.

Bugo aims to be a framework that makes it faster and easier to develop, curate and deploy websites with blogging capability. To make this happen you have to think of a website a little differently. Traditionally a website that had blogging capability had to have a web server, database and a content management system (WordPress, Joomla and others). Bugo uses static site generation to cut that list down to a single web server. Everything else is relegated to a service.

So how does that work?

  1. All your code and content are stored in a git repository like GitHub or BitBucket.
  2. Developers work on the code locally and push their changes to the repository.
  3. Content creators work at with a content management service like Forestry.io or Netlify.com and their content is pushed to the repository.
  4. When the content or code is updated a build is triggered and a static site generator builds a static version of the site.
  5. The static site is then automatically deployed to the web server.

That’s it.

Media Handling

Media can be stored in the repository, but build performance can suffer from a large media library. This can also result in longer deployment times.

Fortunately CMS services like Forestry.io and Netlify.com have thought of this and have integrated with CDNs to make the process of managing Media seamless to the content generator. Developers can still store theme related media in the codebase.

SEO

SEO is not a plugin. Traditional websites rely on plugins to enhance the data they present to google. Not necessary with Bugo. Bugo uses all the same stuff built in. Structured Data, Semantic Layouts, Breadcrumbs & XML Sitemaps

Advantages

There are several advantages to this approach.

  1. Speed. Static sites don’t need a database, so they are inherently fast.
  2. More Secure. Static sites don’t have most of the vulnerabilities a traditional website.
  3. Reduced Sever Loads. Since the server is only serving basic HTML and a limited amount of assets, servers can handle a lot more requests.
